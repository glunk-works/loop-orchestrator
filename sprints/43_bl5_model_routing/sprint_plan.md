### FILEPATH: /sprints/43_bl5_model_routing/sprint_plan.md

**Sprint Goal:** Land **BL-5 per-persona model routing** as the first, most self-contained enabler of the bounty loop's Phase 0 (`docs/bounty_loop_architecture.md` §8) — and the one that also unblocks the paused dev loop. Two things are true today that shape this sprint: (1) per-persona routing **already half-exists** — `GeneratorConfig.model` (`personas/declarative/config.py`) is a per-persona field that flows through `node.py`/`services.py`, and each declarative persona config names its own model; (2) the true blocker is that **`pricing.RATES` contains only `claude-sonnet-5`** — using Opus 4.8 or Haiku 4.5 raises `UnknownModelError` and, by design, **disables the budget cap**. So the bounty personas (Triage=Haiku, Deep-Inspection/Report=Opus) cannot name their model until those rates exist. The deliverable is: **(a)** add `claude-opus-4-8` + `claude-haiku-4-5` to `RATES` with correct list pricing (incl. cache read/write) and pricing tests; **(b)** collapse the **two** duplicated `DEFAULT_MODEL = "claude-sonnet-5"` constants (`personas/coder_iac/shared.py`, `personas/pm/persona.py`) into **one** shared canonical default so there is a single source of the fallback model instead of two copies — a straight de-duplication, **not** a new override/routing mechanism (both call sites are module-level free functions with no persona config in scope, and no persona in this sprint needs to override the default, so per-persona parameterization is deferred until a consumer exists — same YAGNI logic as P0-D2); **(c)** review the resolver `max_tokens: 2048` the backlog flagged (BL-5) and adjust only if warranted. This is Phase 0 **sprint 1 of 3** — sprint 44 (`tools/inventory_db` + the §4 schema) and sprint 45 (the §5 scope validator + §10 ingestion seam) follow. **The dev loop's actual model assignments are NOT changed** (it stays paused); this sprint makes Opus/Haiku *priceable and routable*, not *routed*.

**Out of scope:**
- **Re-routing the dev-loop personas (Architect→Opus, etc.).** BL-5's original backlog intent included assigning Opus to the Architect persona on the *default* loop. That loop is **paused** behind the bounty pivot, so changing its live model assignments would alter behavior on a loop nobody is running (owner-decided this pass: "Enabler + one seam", not "Full BL-5 as backlogged"). The declarative `model:` field makes that a one-line YAML change whenever the dev loop resumes; this sprint deliberately does not make it.
- **Any `State` change / `schema_version` bump.** The `schema_version` 5→6 bump originally listed under Phase 0 is **deferred to Phase 1** (owner-decided this pass) — it ships with the first *bounty* `State` field (the Recon/Inventory stage), not with pure non-`State` infra. Sprint 43 touches no `State` shape.
- **The bounty personas themselves** (Triage/Deep-Inspection/Report). They land in Phases 2–4. Sprint 43 only makes their models *priceable and selectable* — it does not create any bounty `Stage`, `GeneratorConfig`, or prompt.
- **`tools/inventory_db`, the scope validator, the ingestion seam.** Those are sprints 44 and 45 (see the Phase 0 decomposition in Context). Not this sprint.
- **Per-persona override plumbing for the non-declarative call sites, and any generalized model-registry / provider abstraction.** The "one seam" is a modest de-duplication of the default-model constant — **not** threading a `model` parameter through `_run_increment`/`fold_answers` (nothing in this sprint would pass an override; the bounty personas that need Opus/Haiku land in Phases 2–4 as declarative configs, which already carry `model:`), and **not** a new routing framework, config schema, or multi-provider abstraction. The existing `GeneratorConfig.model` field stays the declarative routing mechanism; we do not replace it. When a non-declarative persona genuinely needs a non-default model, parameterizing off the single shared default is a one-line follow-up then.

Defense-in-depth; the strongest fix for `bounty-infra#13`.

**Security Considerations:** Low surface. This sprint adds **no** new subprocess surface (the five sanctioned surfaces are unchanged), **no** new credential class, and **no** untrusted-input path. Diligence items:
1. **Budget-cap integrity is the security-relevant property here.** `UnknownModelError` exists specifically so an unpriced model can't silently bill at $0 and disable the cap (`pricing.py` docstring). Adding Opus/Haiku to `RATES` must use **correct** rates — an under-priced entry would let a run exceed its real `--budget`. The pricing tests (T1) are the guard: assert `cost_usd`/`estimate_cost_usd` compute the expected dollar figures for each new model, and that an unknown model *still* raises.
2. **`extra="forbid"` on `ModelRates` stays intact** — a new entry is a `ModelRates(...)` instance, not a shape change.
3. **The de-duplication must not change *which* model any persona actually uses** (still `claude-sonnet-5`); a regression that silently re-routed the paused dev loop is the failure mode. Cover with a test asserting the canonical default is `claude-sonnet-5` and that both call sites resolve to it.
This sprint **touches `src/`** (`tools/llm/pricing.py`, `personas/coder_iac/*`, `personas/pm/persona.py`, possibly a new small model-default module) ⇒ every such PR needs a fresh-session **`architect-review`**, and the `/critic-gate` pass applies — **architect** (the module-boundary placement of the canonical default; pricing-table correctness) is the indicated critic; **security-critic** is optional here (no taint path, but the budget-cap-integrity angle in item 1 is worth a look if the diff touches the cap math). Docs-only tasks are `architect-review`-exempt.

**Risks & Blockers:**
- **Wrong pricing silently weakens the budget cap.** The single highest-value check. Verify the four numbers per model against the `claude-api` reference (input/output at list; cache-write = input×1.25, cache-read = input×0.1). A typo here doesn't fail a test unless the test asserts the dollar amount — so T1's tests must assert **computed cost**, not just "the key exists".
- **The de-duplication can leak into the paused dev loop's behavior.** Even a pure constant move risks a typo or an import that changes the effective model. Keep the value `claude-sonnet-5`, add a regression test that both call sites resolve to it, and land T2 on its own PR so the behavior-preservation is independently reviewable. **Do not** expand scope into per-call `model` parameters (explicitly deferred) — that is the tempting over-build here.
- **Module-boundary placement of the canonical default.** `core/` imports no concrete persona module (only `personas/base.py`); `tools/llm/client.py` is the sole `keyring` importer. A `tools/llm` constant is the safest home — both non-declarative personas already import `tools/llm` for the client, so no new import edge appears, and it does not touch `core/`'s graph or the `keyring` boundary. Confirm no boundary test regresses and state the chosen home in the PR.
- **`src/`-touching ⇒ `architect-review` on each `src/` PR, in a FRESH session.** Do **not** `/model opus` mid-session to "review". `/handoff` → new session → `/resume` → `/code-review` → post the verbatim-header review.
