**Sprint Goal:** Execute the **host-gated flip block** that ends the migration with **one path, not N** — the tracked teardown of the migration scaffolding described in `docs/migration_roadmap.md` "Phase 6 — Collapse the flags". Concretely: **verify** each new path end-to-end on a daemon-bearing host, then **flip the default and delete the proven-redundant classic path** for the four sunsettable flags (`LOOP_ENGINE_ENGINE=langgraph`, `LOOP_ENGINE_TOOLS=mcp`, `LOOP_ENGINE_PERSONAS=declarative`, `LOOP_ENGINE_CODER=ralph`), strip the dual-field `artifacts`/`artifact_refs` from `State` (downstream of `run_loop` deletion; bump `schema_version` 3→4 + extend `migrate_state_payload`), collapse the `loops/default/loop.py` flag branches to the single surviving wiring, and flip the issue path onto MCP while landing Sprint 26's HITL findings **R1–R7**. `LOOP_ENGINE_ISOLATION` **stays** (genuine runtime config, not scaffolding). The block ends with the migration's flags gone, `State` truly thin, and `sprints/DEFERRED_VERIFICATION.md` deleted.

**⚠ HOST-GATED — this sprint does NOT start on a laptop.** Every deletion is gated on its replacement being **live-verified on a daemon-bearing host** (the same host the deferred 3b/Ralph/§5–§9 checks need: real `gh` auth, a container runtime, a real Anthropic key, network). The verification runs (V1–V3 below) consume **real API budget** and have **real GitHub side effects** — they are human-operated, not part of the green gate. Implementation (the code deletions) is mechanical once verification passes, but **no deletion task may land before its gating verification is recorded as PASSED** in `DEFERRED_VERIFICATION.md`. If a host session cannot complete all of V1–V3 + all deletions in one sitting, split along the task boundaries below — each task is its own independently-committable, green, reviewable change (per the phase's "Discipline" rule).

**Out of scope:** any *feature* work (this block only removes scaffolding — no new behavior); `LOOP_ENGINE_ISOLATION` (kept — it is `none` for local dev / `container` for the factory host, genuine config, never old-vs-new); the Phase 3b container/sandbox launch mechanism (not a flag, not scaffolding — it is how the MCP path runs on the host and stays); introducing any new flag or dependency (a phase whose purpose is *removing* optionality must not add any).
