**Sprint-45 specifics (this plan):**

- **`tools/scope_validator` — the structural scope validator (§5; P0-D12/D13/D14).** A new pure leaf module:
  - `ScopeRules` — a **frozen** Pydantic model (`extra="forbid"`): `in_scope_regex: list[str]`, `out_of_scope_regex: list[str]`, `banned_actions: list[str]`. A `@classmethod from_target(source) -> ScopeRules` reads `.in_scope_regex`/`.out_of_scope_regex`/`.banned_actions` off a **structural protocol** (`_HasRulesOfEngagement`), with `Target` imported only under `TYPE_CHECKING` — **no runtime edge** onto `inventory_db` (P0-D12). Regex patterns are **compiled once** at construction (a `model_validator`/cached property), so an invalid pattern is a construction-time error surfaced clearly, not a per-call surprise.
  - `validate_target(rules: ScopeRules, candidate: str) -> None` — the fail-closed allowlist (P0-D13): raises `ScopeViolation(candidate, reason)` when the candidate matches 0 `in_scope_regex` **or** matches any `out_of_scope_regex` (deny wins). Returns `None` on allow. Uses `re.search` on the pre-compiled patterns; the reason string names *which* rule vetoed (out-of-scope hit vs. no in-scope match) **without** echoing attacker-influenceable candidate content into logs beyond the candidate already passed in.
  - `is_action_banned(rules: ScopeRules, action: str) -> bool` — the pure classifier (P0-D14). No raise, no escalation — the caller owns policy (§6).
  - `ScopeViolation(Exception)` — a typed error carrying the candidate + reason, so a Phase-1 caller can surface it as a gate/escalation signal (§5).
- **`tools/ingest` — the ingestion-sanitization seam (§10; P0-D6/D15).** A new pure leaf module:
  - `sanitize(text: str, *, max_len: int) -> str` — the structural normalizer (P0-D15): strips C0/C1 control characters, ANSI CSI escape sequences, and zero-width/BOM code points; applies Unicode NFKC normalization; collapses whitespace runs; then hard-truncates to `max_len` (the per-field cap that bounds both context cost and injection-payload size). Deterministic and pure — identical input ⇒ identical output. A module-level default cap constant documents the intended per-field bound; no phrase blocklist (P0-D15).
- **Module boundaries (two new leaf modules; static-test-enforced).** Both `tools/scope_validator` and `tools/ingest`:
  - import **no `keyring`** (the keyring boundary test stays green), **no `psycopg`** (the sprint-44 psycopg boundary test stays green — these modules are not permitted importers), and **no `subprocess`** — the **five** sanctioned subprocess surfaces in `tests/tools/test_subprocess_surfaces.py` are **unchanged**; this adds no sixth.
  - **write no files** — they are pure functions over in-memory values; nothing goes through/around the `state_io`/`scaffold` write surfaces.
  - **`scope_validator` imports nothing from other `tools/*`** at runtime (P0-D12) — pin this with a focused import-graph assertion (no runtime `inventory_db` edge), the standalone-leaf analog of the existing boundary tests.
  - No existing module imports either package this sprint (no consumer, P0-D11), so `core/`'s import graph is untouched.

**Security Considerations:** This sprint **is** the security work — it builds the two structural invariants that gate all attacker-influenceable input in Phase 1+, but with **no untrusted data flowing yet** (no consumer — P0-D11). Diligence items:
1. **Fail-closed is the invariant.** The scope decision is an **allowlist** (P0-D13): empty/absent `in_scope_regex` denies everything; an out-of-scope match always vetoes. Tests must pin the fail-closed edges explicitly — empty rules deny, out-of-scope-beats-in-scope, and a violation *raises* (never returns / silently no-ops). This is the §5 analog of `_validate_clone_dest`/no-merge-verb/fixed-argv: the model can never widen scope because the primitive refuses, in code, before anything runs.
2. **The sanitizer must be robust, not clever (P0-D15).** Structural normalization only. Tests pin control-char/ANSI/zero-width removal, NFKC folding, and the hard length cap on adversarial inputs (embedded escapes, homoglyphs, over-long banners). No phrase blocklist to bypass. It builds the injection-resistant seam *now*, before Phase-1 scanner text flows (§10), exactly as sprint 44 built the parameterized-SQL seam before any scanner data reached the DB.
3. **No new credential class, no new sink, no new subprocess surface.** Pure functions; the five subprocess surfaces and every credential boundary are unchanged. The one thing to watch is not leaking attacker-influenceable candidate/scanner text into logs beyond what the caller already holds — error/reason strings stay structural.
4. **`extra="forbid"` + frozen on `ScopeRules`** — the rules-of-engagement is a fixed, immutable shape, not an open bag; a typo'd rule field is a construction error, and rules can't be mutated after validation.
5. **`regex` denial-of-service posture.** Patterns come from the *operator's* `targets` rows (trusted rules-of-engagement, not attacker input), and are compiled once at `ScopeRules` construction so a bad pattern fails fast at the seam. Note this trust boundary in the plan; catastrophic-backtracking hardening on operator-authored patterns is not in scope this sprint (no untrusted pattern source exists).

**Risks & Blockers:**
- **A fail-*open* scope default silently defeats the whole invariant.** The single highest-value check. If empty `in_scope_regex` ever allows, or an out-of-scope match ever loses to an in-scope match, §5 is void. Pin both edges with explicit tests; treat any "allow when unsure" as a bug.
- **An incomplete sanitizer gives false confidence.** Missing a control-char class or the length cap means malformed/oversized scanner text still reaches a model in Phase 1. Test against adversarial inputs (ANSI escapes, zero-width, homoglyphs, over-length); NFKC + the cap are not optional.
- **The standalone-leaf property must actually hold (P0-D12).** `scope_validator` must have **no runtime import edge** onto `inventory_db` — the `Target` type reference is `TYPE_CHECKING`-only. Verify with the import-graph assertion; a leaked runtime import re-couples the leaf.
- **The boundary guards must actually constrain (BL-15).** Any new structural guard (the no-runtime-edge assertion) lands in the **same T1 PR** as the code it guards; if guard-adversary is run, confirm it fails under a planted violation — a vacuous guard is worthless.
- **`src/`-touching ⇒ `architect-review` on the T1 PR, in a FRESH session.** Do **not** `/model opus` mid-session to "review". `/handoff` → new session → `/resume` → `/code-review` → post the **verbatim-header** review. Watch the **BL-35 stale-red** trap (BLOCKED + rollup FAILURE ⇒ `gh run rerun` the OLD run — `architect-review` fires on both `pull_request` and `pull_request_review`).
- **Full local gate before push.** Run lint → format → test (or `/ship`) — two new packages are exactly the diff that trips `format-check`/`ruff` if only `hatch run test` was run (memory: #141). **PR title ≤72 bytes** — `wc -c` first. **Never commit to `main`, merge, or force-push.** No `sbom`/`audit` action needed (no new dep), but confirm `hatch run audit`/`sbom` stay green incidentally.
