**Context (locked decisions — do not re-open):**

- **FD1 — Prompt-only fix, no gate-guard** *(locked 2026-07-11, Opus/Architect, F-RALPH-OVERSPEC-TEST planning; repo-owner-confirmed)*. The run-#6 escalation was **model-initiated**: Ralph emitted an `## Open Questions` section (extracted by `_new_questions`/`extract_open_questions`) about a test it had itself written. The direct lever is therefore the prompt, not the gate. A gate-level guard (red gate whose only failing tests are self-authored + out-of-spec → repair increment instead of human) would be more robust but requires reliably classifying a test as "self-authored and out-of-spec," which is brittle and a larger, more speculative diff. Prompt-first is low-risk, directly targets the observed failure, and is re-verifiable on the next host V2 run. If a prompt-only fix proves insufficient on that re-run, the gate-guard becomes a follow-on — not this sprint.
- **FD2 — The two guardrails are additive prompt directives, not a scope renegotiation.** Ralph's system prompt already forbids guessing and mandates escalation for genuine ambiguity (`PROMPT_TEMPLATE` directive 3, `shared.py`). The fix **narrows what counts as an escalation-worthy ambiguity** (spec ambiguities only — never a self-authored test you can correct) and **bounds test authorship** (cover the acceptance criteria's enumerated cases; do not assert private internals / import mechanics / behavior beyond spec). It does not touch directive 1 ("implement only this task") for *production* code — the self-fix carve-out is scoped to **tests the coder itself authored**, so it cannot become a backdoor to cross-task feature work.

**Security Considerations:** Prompt-text-only change to model-facing instructions. Sanctioned subprocess surfaces stay **exactly five** (unchanged); `tools/llm/client.py` stays the only `keyring` importer; the three MCP servers' tool-sets stay pairwise disjoint; `State`/`schema_version`/`extra="forbid"` untouched. No new file-write, network, or credential path. The guardrail must not weaken the *security* content of `PROMPT_TEMPLATE` (the Global-DoD / no-secrets directives) — it is strictly additive.
