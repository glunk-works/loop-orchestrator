**Sprint Goal:** Stand up the **bounty `Loop` skeleton** and land the one irreversible change in Phase 1 — the **`State.schema_version` 5→6 bump** — in the smallest, most reviewable unit, *before* any external surface exists. This sprint creates `loops/bounty/loop.py::build_bounty_loop()` wiring **both** Phase-1 stages (Recon/Inventory → Surface-Mapping) with `impact_reentry={"scope": 0, "surface": 1}`, adds the first *bounty* `State` field as a nested `BountyRunState` sub-model (`target_id`), bumps `CURRENT_SCHEMA_VERSION` to 6 with the matching `migrate_state_payload` branch, and drives the whole loop **green end-to-end hermetically** through the engine with **stub personas** that emit fixture artifacts. This is Phase 1 (Recon + Surface-Mapping) **sprint 1 of 3** (P1-D1): **46 (this plan — skeleton + State bump)** → 47 (recon data path: `workflow_dispatch`/S3/IDP → `inventory_db`) → 48 (Surface-Mapping stage). The stub persona **bodies** are replaced behind a stable seam in S47/S48; the loop wiring, the gates, the re-entry map, and the schema shape land **now** and do not churn again (P1-D3). This sprint is the **first `State`-touching PR** of the whole bounty initiative — all of Phase 0 was pure non-`State` infra — so it carries the fresh-session `architect-review` on the schema change.

**Out of scope:**
- **Any real recon/scan surface.** No `workflow_dispatch` trigger, no S3 fetch, no `boto3`, no scanning MCP tool/server, no `mcp_servers/` addition. The stub personas emit **fixture** artifacts; the real dispatch→S3→IDP→`inventory_db` data path is **S47** (P1-D4/D5). Building it here — with no schema bump to anchor it and no stage shell to host it — is exactly the speculative-surface trap P0-D11/D7 rejected.
- **Any `inventory_db` consumer / live Postgres.** S46 opens no DB connection and does not discharge the OWED sprint-44 live PG round-trip smoke (`DEFERRED_VERIFICATION.md` §10). That discharge is **S47**, folded into the single authorized V-run alongside recon's own live smoke (P1-D4). Do **not** stamp §10 verified this sprint.
- **The scope validator / ingestion sanitizer as live consumers.** `tools/scope_validator` and `tools/ingest` gain **no caller** in S46 (they stay the pure Phase-0 leaves). Their first live mount — `validate_target` raising on caller input, and the discovered-asset output filter + `ingest.sanitize` — is **S47** at the recon tool's Pydantic boundary (P1-D6). Stub personas produce trusted fixtures, so no untrusted input flows yet.
- **CLI / runner loop-selection.** No `--loop {default,bounty}` flag, no `bounty-run` subcommand (P1-D7). The bounty loop is a **library `Loop`** in S46, driven through the engine in tests; the CLI selector lands in **S47**, when there is a real recon path worth invoking by hand. Adding a human entrypoint for a stub-only loop is YAGNI.
- **Real persona logic — LLM calls, dispatch, parsing, DB writes.** The S46 personas are **stubs**: they read `State.bounty`, delegate to an injected producer collaborator (default = a fixture stub), write the stage artifact, and return `State`. No `LLMClient` call, no network, no subprocess. S47/S48 inject the real collaborators behind the same seam.

Deferring this would force the gate wiring to change in S47, which is exactly the churn **P1-D3** forbids. Consequence for S46: the stub bodies below must be **valid JSON of the matching type** (an empty inventory is the JSON `[]`; `require_nonempty_parse` is left **False** so the stub's empty-but-valid inventory ACCEPTs). *(If the owner expects a different real shape for either artifact, change it here before T1 — it is the one wiring value that must not move after S46.)*

  - **The collaborator protocol — `personas/bounty/producers.py`.** A `Protocol` `ArtifactProducer` with **`def __call__(self, bounty: BountyRunState) -> str: ...`** — it takes the (already-non-`None`) `BountyRunState` and returns the **artifact body as a `str`** (because `State.artifacts` is `dict[str, str]`; a JSON producer returns `json.dumps(...)`). It does **not** receive `state`, `llm_client`, or `findings` — the persona shell owns all of those. Ship two default **fixture-stub** producers here (one per stage). *(Locked: `producers.py` is the home — not "if cleaner".)*
  - `ReconPersona(BasePersona)` — `produces = ("asset_inventory",)`, `consumes = ()`. **The shell owns:** (1) read `state.bounty` and **raise a clear error if `state.bounty is None`** — a bounty run without a target is a misconfiguration, not a silent no-op; note `target_id` is **not** an artifact, so the engine's `consumes` pre-check never covers it — this raise is its *only* guard (do **not** route `target_id` through `consumes`); (2) call the injected `ArtifactProducer(bounty)` to get the body; (3) return `state.model_copy(update={"artifacts": {**state.artifacts, "asset_inventory": body}})` — the immutable-update idiom used by every persona (see `RecordingPersona` in `tests/core/test_graph_engine.py`), never in-place mutation. `findings` is accepted (the `run` signature requires it) and **ignored** by the stub — it always reproduces the same body. No `LLMClient` use. **S47 injects the real dispatch→S3→IDP→`inventory_db` producer behind this exact `__call__` seam.**
  - `SurfaceMapPersona(BasePersona)` — `consumes = ("asset_inventory",)`, `produces = ("surface_map",)`. Same shell shape and same `model_copy` return; default stub producer returns the `surface_map` body. **S48 injects the real mapping producer.**

Rationale: on a REVISE the engine re-runs the stub, which reproduces the identical artifact, so `gate_result.findings == previous_gate_findings` fires the exhaustion-escalation path (`engine.execute_stage`) — the run would pause, not stay green. A JSON body matching the gate type ACCEPTs first try and avoids this entirely.

**Security Considerations:** S46 introduces **no untrusted-input path** — the stub personas emit trusted fixtures, no scanner/target text flows, no network, no subprocess. The security-relevant surface is entirely the **schema change** and the **shape of the seam** the real (untrusted) data path will mount into in S47:
1. **`extra="forbid"` and the migrate branch must both hold.** `BountyRunState` and the extended `State` keep `extra="forbid"`; a v5 snapshot must upgrade to v6 cleanly (additive `bounty=None`) and a v6 snapshot round-trip through `migrate_state_payload` unchanged. An unsupported version still raises. Pin all three: v5→v6 upgrade, v6 identity, and an unknown version raising.
2. **The Recon persona fails closed on a missing target.** `state.bounty is None` (or an absent `target_id`) **raises**, never silently produces an empty inventory — the §5 fail-closed ethos applied at the loop's entry, before the real scope validator exists. A bounty run with no rules-of-engagement root must not proceed.
3. **The seam must not let S47's untrusted path bypass a boundary.** The injected-collaborator seam is where S47 mounts `scope_validator`/`ingest` (P1-D6). Design it so the collaborator returns *already-scope-filtered, already-sanitized* data and the persona shell is the single writer of `state.artifacts` — no path around the (future) validator. Note this intent in the plan; the enforcement lands with the consumer in S47.
4. **No new credential class, no new sink, no new subprocess surface.** Confirmed: `boto3`/AWS-OIDC and the `gh` `workflow_dispatch` consumer are S47 (P1-D5); S46 adds none.
