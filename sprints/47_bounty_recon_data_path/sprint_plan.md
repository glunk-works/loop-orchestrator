**Security Considerations:** S47 is where **untrusted input first flows** (raw scanner output + target-derived text — §10). The design *is* the mitigation:
1. **Both scope boundaries live and complete (P1-D6).** Input: `validate_target` **raises** on an out-of-scope seed before any `gh` dispatch. Output: every discovered identifier is checked with `is_in_scope`; out-of-scope ones **drop+count**; endpoints inherit host-level scope (S47-D11). T2's review must confirm **both** are present.
2. **Anti-evasion normalization (S47-D5).** Scope is decided on the *normalized* identifier and that form is stored — invisible-char/homoglyph hosts can't slip past `re.search`. Fixture case (c) pins it.
3. **Sanitize before persistence, before any model (bounty-infra#13 fix).** `matched-at`/banners/`tech_stack` are `sanitize`d before `inventory_db`. Case (d) pins it.
4. **Fail-closed on parse.** Malformed batch raises; bad record drops+counts. Case (e) pins it. Never a silent no-op (§5).
5. **`gh` argv is a dangerous sink (§10).** Fixed argv, `shell=False`, no model-controlled flag; the seed passed to `workflow_dispatch` has already cleared `validate_target`. No sixth subprocess surface.
6. **The producer never re-dispatches on a gate re-run.** The `parse_json="list"` gate ACCEPTs any valid list first-try, so the Recon stage does not REVISE and the producer is not re-invoked — no double Fargate spend. (If a future gate could REVISE, dispatch must be made idempotent per correlation token; noted, not needed now.)
7. **No new credential class.** `boto3`/AWS-OIDC + the `gh` consumer ride already-declared classes; the Anthropic key stays keyring-only; `s3_io` is the sole `boto3` importer; nothing new imports `keyring`.
