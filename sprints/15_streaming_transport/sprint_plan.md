### FILEPATH: /sprints/15_streaming_transport/sprint_plan.md

**Sprint Goal:** Migrate the LLM client's sole transport call from non-streaming `messages.create` to `messages.stream(...)` + `get_final_message()`, remove the timeout override that existed only to dodge the SDK's non-streaming size guard, and fail fast on `stop_reason == "refusal"`.

**Dependencies:** None. (Sprints 16 and 17 build on this one.)

**Security Considerations:** No new modules; `tools/llm/client.py` remains the only module importing `keyring` (`tests/tools/test_keyring_boundary.py`) and performs no file I/O (`tests/tools/test_state_io_boundary.py`). The `anthropic.Anthropic(api_key=api_key)` construction must stay shape-compatible: `tests/tools/test_llm_client_tls.py` asserts `client._anthropic._client._transport._pool._ssl_context.verify_mode == ssl.CERT_REQUIRED` against the real SDK client and greps the source for TLS-disabling patterns — do not add `verify=`/`CERT_NONE` anywhere. No dependency changes (`anthropic==0.116.0` already ships `messages.stream`), so `sbom.json` and `hatch run audit` are untouched.

**Risks & Blockers:** The blast radius is almost entirely in tests, not source: 19 references across `tests/tools/test_llm_client.py` plus the two integration tests mock `_anthropic.messages.create` directly and assert exact transport call counts. Build ONE shared mock helper and migrate every mock through it — do not hand-roll per-test context managers, and keep the exact-call-count discipline (`messages.stream.call_count`) intact. `run_tool_loop` feeds `response.content` (SDK block objects) back as assistant content; `get_final_message()` returns the same `Message` shape as `create()` (same `usage`, `stop_reason`, `content`), so downstream logic must not change — if a task seems to require changing persona code, the task is being done wrong. Rationale for the timeout revert: the 2400s override existed solely because the SDK refuses non-streaming requests it estimates will exceed ~10 minutes at high `max_tokens`; streaming keeps the connection active, so the guard no longer applies and the SDK default timeout is correct again.

**Tasks:**

- **Task 1: Stream the transport call**
  - **Description:** In `src/loop_engine/tools/llm/client.py` `_request()`, replace `response = self._anthropic.messages.create(model=..., max_tokens=..., messages=..., **request_kwargs)` with `with self._anthropic.messages.stream(model=..., max_tokens=..., messages=..., **request_kwargs) as stream:` / `response = stream.get_final_message()`. Everything after (usage reads via `getattr(..., 0) or 0`, cost debit, `max_tokens` truncation check, text join, tuple return) is byte-for-byte unchanged. In `__init__`, revert `anthropic.Anthropic(api_key=api_key, timeout=2400.0)` to `anthropic.Anthropic(api_key=api_key)` and delete the timeout comment block (streaming removes the reason it existed). Update the module docstring/comments that say "sends" a request to note the transport streams and accumulates to a final message.
  - **Target Files:** `src/loop_engine/tools/llm/client.py`
  - **Acceptance Criteria:** No persona file changes. `messages.create` appears nowhere in `src/`. `tests/tools/test_llm_client_tls.py` passes unchanged. `hatch run lint` passes.

- **Task 2: Refusal fails the stage honestly**
  - **Description:** Add `class RefusalError(Exception)` to `src/loop_engine/tools/llm/client.py` (docstring: a refusal returns empty/partial content that would otherwise reach the gate as a REVISE and burn paid re-rolls; re-rolling the same prompt is not a fix). In `_request()`, after the cost debit (tokens spent are real and must be accounted), raise it when `getattr(response, "stop_reason", None) == "refusal"` — mirror the existing `max_tokens` check. In `src/loop_engine/core/engine.py`, extend the existing `except TruncatedResponseError` branch (the one that `_finalize(state, stage_index, RunStatus.FAILED_STAGE)`s then raises `InvalidStateTransitionError`) to also catch `RefusalError` with a message naming the stage and the refusal.
  - **Target Files:** `src/loop_engine/tools/llm/client.py`, `src/loop_engine/core/engine.py`, `tests/tools/test_llm_client.py`, `tests/core/test_engine.py`
  - **Acceptance Criteria:** A client test asserts a `stop_reason="refusal"` response raises `RefusalError` with tokens/cost still debited (mirror `test_truncated_response_raises_instead_of_returning_partial_text`). An engine test asserts a persona raising `RefusalError` produces a `FAILED_STAGE` snapshot and `InvalidStateTransitionError` (mirror the existing truncation engine test).

- **Task 3: Migrate transport mocks through one shared helper**
  - **Description:** Add a `_mock_stream_transport(*responses)` helper (module-level in `tests/tools/test_llm_client.py`; duplicated verbatim or imported per repo test conventions in the two integration tests): builds a `MagicMock` transport whose `messages.stream(...)` returns a context manager (`MagicMock` with `__enter__` returning an object whose `get_final_message()` pops the next `SimpleNamespace` response; `side_effect`-style ordering preserved). Migrate every `messages.create` mock in `tests/tools/test_llm_client.py`, `tests/integration/test_budget_abort.py`, and `tests/integration/test_no_credential_leakage.py`; every `messages.create.call_count` / `call_args` assertion becomes the `messages.stream` equivalent (request kwargs are now `stream.call_args.kwargs` — the system/messages/tools assertions carry over unchanged). Keep `count_tokens` stubs as-is (`count_tokens` is not streamed).
  - **Target Files:** `tests/tools/test_llm_client.py`, `tests/integration/test_budget_abort.py`, `tests/integration/test_no_credential_leakage.py`
  - **Acceptance Criteria:** `hatch run test` fully green. Every prior exact-call-count assertion has a `messages.stream` counterpart (e.g. budget-abort still proves exactly 2 transport calls; the tool-loop iteration-cap test still proves exactly 3). No test constructs a raw stream mock outside the helper.

- **Task 4: Documentation sweep**
  - **Description:** Update `CLAUDE.md`'s `tools/llm/client.py` bullet: transport streams every call via `messages.stream` + `get_final_message()` (the SDK-recommended shape for large `max_tokens`), and `stop_reason == "refusal"` raises `RefusalError` (fail-fast, like truncation). Check `README.md` and `docs/architecture_definition.md` for any mention of `messages.create`, the request timeout, or "non-streaming" and align them.
  - **Target Files:** `CLAUDE.md`, `README.md`, `docs/architecture_definition.md`
  - **Acceptance Criteria:** No document describes the transport as non-streaming or mentions the timeout override. `hatch run test`, `hatch run lint`, `hatch run format` pass.
